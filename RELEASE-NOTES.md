# SPSWakeUp - Release Notes

# [4.2.1] - 2026-05-06

### Added

- Add new PowerShell 7 worker script: `scripts/SPSWakeUp-pwsh.ps1`.
- Add new Pester test suite for the PS7 worker script: `tests/SPSWakeUp-pwsh.Tests.ps1`.
- Add hybrid execution model:
  - `SPSWakeUP.ps1` orchestrates install/uninstall, SharePoint URL collection, and compatibility logic.
  - `SPSWakeUp-pwsh.ps1` executes warm-up web requests with PowerShell 7 features.
- Add automatic fallback to Windows PowerShell 5.1 warm-up flow when PowerShell 7 (`pwsh`) is not available.
- Add JSON handoff file (`SPSWakeUp_urls.json`) generated by `SPSWakeUP.ps1` and consumed by `SPSWakeUp-pwsh.ps1`.
- Add PowerShell version output in warm-up summary reports for both script flows.
- Add PSGallery publishing of both scripts in `.github/workflows/release.yml`.

### Changed

SPSWakeUP.ps1:

- Update script version metadata and in-script version variables to `4.2.0`.
- Rework `Default` and `AdminSitesOnly` actions to:
  - collect SharePoint URLs in the main script,
  - launch `SPSWakeUp-pwsh.ps1` when PowerShell 7 is available,
  - fallback to the existing PowerShell 5.1 warm-up functions otherwise.
- Load SharePoint context for all actions requiring SharePoint cmdlets (`Install`, `Uninstall`, `Default`, `AdminSitesOnly`).

Documentation:

- Update `README.md` with two-script architecture and PowerShell Gallery installation guidance for both scripts.
- Add optional `PowerShell 7.x` prerequisite section in `README.md` with official Microsoft installation source URL.
- Update wiki pages (`Home.md`, `Features.md`, `Getting-Started.md`, `Usage.md`) to reflect hybrid execution and fallback behavior.
- Update `tests/README.md` with PS7 test execution examples and code coverage paths for both scripts.

Tests/CI:

- Update `tests/SPSWakeUP.Tests.ps1` to assert companion script presence (`SPSWakeUp-pwsh.ps1`).
- Update `.github/workflows/pester.yml` to run `PSScriptAnalyzer` on the full `scripts` folder (both scripts).

### Fixed

SPSWakeUP.ps1:

- Fix null-valued expression warning during install path (`You cannot call a method on a null-valued expression`) by ensuring `hostEntries` is initialized when `Get-SPSWebAppUrl` is called outside `Invoke-SPSAllSite`.
- Replace PowerShell 7-only null-conditional syntax in orchestrator path with PowerShell 5.1-compatible command resolution logic for `pwsh` discovery.

SPSWakeUp-pwsh.ps1:

- Add `-SkipCertificateCheck` for PowerShell 7 web requests in warm-up/authentication paths.
- Add `-AllowUnencryptedAuthentication` where `UseDefaultCredentials` is used against HTTP endpoints to prevent PS7 authentication blocking.
- Refactor `Invoke-WebRequest` calls to use splatting and add `Get-SPSAuthWebSession` helper for authentication session bootstrap.
- Replace non-ASCII dashes in warning messages with ASCII characters to satisfy analyzer encoding checks.
- Fix `PSScriptInfo` GUID format to a valid GUID value for script publishing compatibility.

Code quality:

- Resolve `PSUseDeclaredVarsMoreThanAssignments` warning in `SPSWakeUP.ps1` by switching local `hostEntries` initialization to `Set-Variable` in `Get-SPSWebAppUrl`.
- Run `PSScriptAnalyzer` with no remaining Error/Warning findings on the `scripts` folder.
